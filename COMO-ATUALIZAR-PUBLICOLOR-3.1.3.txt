PUBLICOLOR PCP 3.1.3 — MATERIAIS E COMPRAS AUTOMÁTICAS

1. Faça backup do repositório e do Supabase.
2. No Supabase de homologação, execute:
   SQL-ATUALIZACAO-PUBLICOLOR-3.1.3.sql
3. Substitua os arquivos do projeto pelo conteúdo deste ZIP, preservando apenas:
   - .git
   - .env.local
4. Execute:
   corepack enable
   pnpm install --frozen-lockfile
   pnpm validate
5. Teste o fluxo completo:
   - abra uma OS;
   - adicione um material como Não disponível;
   - confirme a atividade no grupo Compras;
   - altere os status na ordem correta;
   - finalize a atividade;
   - confirme que o material ficou Disponível na OS;
   - reabra a atividade e confirme que o material voltou para Não disponível.
6. Após a homologação, execute o mesmo SQL no Supabase de produção.
7. Envie o código:
   git add -A
   git commit -m "Publicolor 3.1.3: materiais e compras automáticas"
   git push

IMPORTANTE
- O prazo da atividade automática é exatamente 24 horas após sua criação.
- O responsável é o usuário que cadastrou ou marcou o material como não disponível.
- O grupo Compras é reutilizado; se não existir, será criado automaticamente.
- Atividades finalizadas ficam ocultas por padrão.
- Não execute novamente todas as migrations antigas.
