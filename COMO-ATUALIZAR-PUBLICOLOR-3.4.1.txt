PUBLICOLOR PCP 3.4.1 — ORDEM DE ATUALIZAÇÃO

ATENÇÃO
- O banco conectado auditado ainda não possui as colunas completas de Materiais e Compras.
- Execute o SQL cumulativo 3.4.1 antes de publicar o novo código.
- Faça primeiro em homologação.

1. SUPABASE DE HOMOLOGAÇÃO

Execute:
SQL-ATUALIZACAO-PUBLICOLOR-3.4.1.sql

Depois execute:
SQL-VALIDAR-PUBLICOLOR-3.4.1.sql

Confirme que a validação informa as colunas de disponibilidade, compra, preços e recebimento.

2. PROJETO LOCAL

Preserve:
.git
.env.local

Substitua os demais arquivos pelo conteúdo do ZIP na raiz do repositório.

3. VALIDAÇÃO LOCAL

Use Node.js 24.x:

corepack enable
pnpm install --frozen-lockfile
pnpm validate

4. TESTE FUNCIONAL

- Abra uma OS e entre em Materiais.
- Edite nome, quantidade, unidade, disponibilidade, preço e recebimento.
- Confirme a alteração em Atividades e Compras.
- Edite o mesmo item pela atividade e confirme a alteração na OS.
- Teste a Agenda em mês, semana e dia.
- Teste em celular, tablet e computador.

5. PRODUÇÃO

Após homologar:
- execute SQL-ATUALIZACAO-PUBLICOLOR-3.4.1.sql no Supabase de produção;
- execute SQL-VALIDAR-PUBLICOLOR-3.4.1.sql;
- publique o código.

git add -A
git commit -m "Publicolor 3.4.1: materiais editaveis e agenda responsiva"
git push

6. PWA

Após o deploy, aceite o aviso de nova versão. No computador, use Ctrl + Shift + R caso o navegador ainda mostre a interface antiga.
