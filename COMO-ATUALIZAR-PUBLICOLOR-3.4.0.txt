PUBLICOLOR PCP 3.4.0 — ORDEM DE ATUALIZAÇÃO

IMPORTANTE
- O Dashboard permanece como página inicial.
- Esta versão não possui módulo de fornecedores ou cotações.
- O SQL 3.4.0 é cumulativo e deve ser executado antes de publicar o código.
- Não execute separadamente os SQLs antigos 3.1.3, 3.1.4 ou 3.2.0 depois do SQL 3.4.0.

1. HOMOLOGAÇÃO — BANCO

Execute no Supabase de homologação:

SQL-ATUALIZACAO-PUBLICOLOR-3.4.0.sql

Depois execute:

SQL-VALIDAR-PUBLICOLOR-3.4.0.sql

O arquivo de validação deve informar as estruturas obrigatórias e não deve apontar atividades ou materiais órfãos.

2. PROJETO LOCAL

Preserve somente:

.git
.env.local

Substitua os demais arquivos pelo conteúdo do ZIP, diretamente na raiz do repositório.

3. VALIDAÇÃO LOCAL

corepack enable
pnpm install --frozen-lockfile
pnpm validate

4. TESTES FUNCIONAIS EM HOMOLOGAÇÃO

- login e Dashboard;
- atualização da página em cada rota;
- Kanban no computador, tablet e celular;
- filtros e ações em lote dos pedidos;
- paginação e reabertura dos concluídos;
- Agenda em mês, semana e dia;
- atividade principal e subatividades;
- edição do material pela atividade e conferência na OS;
- recebimento parcial e valores de compra;
- clientes, usuários e configurações;
- miniaturas, Google Drive e diagnóstico;
- atualização do PWA preservando a rota.

5. PRODUÇÃO — BANCO

Depois de aprovar a homologação, execute no Supabase de produção:

SQL-ATUALIZACAO-PUBLICOLOR-3.4.0.sql
SQL-VALIDAR-PUBLICOLOR-3.4.0.sql

6. PUBLICAÇÃO

git add -A
git commit -m "Publicolor 3.4.0: reforma completa e otimizacoes"
git push

7. APÓS O DEPLOY

No computador:
Ctrl + Shift + R

No celular ou tablet:
- feche o PWA;
- abra novamente;
- use o aviso "Nova versão disponível" quando apresentado.

O sistema deve permanecer na mesma página durante a atualização. A entrada de uma nova sessão continua sendo /dashboard.
